float ic::math::vector2_t::Cross2D ( const vector2_t &  Other) const
inline

Given a vector v = <x1, y1> and a vector w = <x2, y2>, their cross-product is determined as <0, 0, x1*y2 - y1*x2>. So, this method returns the third component.

This value can be used to determine which side of a vector another vector is on. If the return value is negative, the "Other" vector is on the left (going ccw). If positive, the vector is on the right (going c/w).

Parameters:
vector2_t&Vector to test x-product on
Returns:
2D cross product (z-component of 3D cross).

void ic::math::vector2_t::Rotate ( const float  radians)
inline

The rotation matrix (in right-hand Cartesian plane) is defined as being | x | | cos(d), -sin(d) | | y | | sin(d), cos(d) |

But in the OpenGL coordinate system, the origin is in the top-left, as opposed to bottom-left, as shown above. So rotations are actually inverted and the matrix is | x | | cos(d), sin(d) | | y | | -sin(d), cos(d) |

So the final rotation in OpenGL would be: x = x * cos(d) + y * sin(d) y = -x * sin(d) + y * cos(d)

floatThe rotation angle in radians.
